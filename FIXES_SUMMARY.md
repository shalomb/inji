## Notes for Lisa (Architecture/Planning)

The refactoring maintains the same logical structure (Jinja2 CLI + globals + tests) but modernizes the build/release infrastructure:

- **Before:** setup.py with custom commands, venv-based workflow, no CI
- **After:** pyproject.toml (PEP 517), uv-based workflow, full GitHub Actions CI/CD

Key architectural decisions made:
- Single entry point via [project.scripts] (no dual paths)
- Pre-declared dependencies for tools (no CI mutations)
- Consistent Python 3.12+ floor across all configs
- Reproducible builds via committed uv.lock
